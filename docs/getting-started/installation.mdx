# Installation Guide

Welcome to the installation guide for forking the application from GitHub and setting up your development environment!

<Steps>
  <Step>
    <StepTitle>Step 1: Fork the Repository</StepTitle>
    <StepContent>
    <div>
      <div>
        Visit our GitHub repository and click the "Fork" button to create your own copy.
      </div>
        <a
          href="https://github.com/sanjayc208/pinedocs"
          target="_blank"
          rel="noopener noreferrer"
        >
          Fork the Repository on GitHub
        </a>
      </div>
    </StepContent>
  </Step>
  <Step>
    <StepTitle>Step 2: Clone Your Fork</StepTitle>
    <StepContent>
    <div>
      After forking, clone your repository to your local machine:
  ```
  git clone https://github.com/sanjayc208/pinedocs.git
  ```
      </div>
    </StepContent>
  </Step>
  <Step>
    <StepTitle>Step 3: Install Dependencies</StepTitle>
    <StepContent>
    <div>
      Install the project dependencies using your preferred package manager:
       <CodeTabs tabs={{
        'npm':{
          syntax:'npm i'
        },
        'pnpm':{
          syntax:'pnpm i'
        },
        'yarn':{
          syntax:'yarn i'
        },
       }} />
      </div>
    </StepContent>
  </Step>
  <Step>
    <StepTitle>Step 4: Run the Development Server</StepTitle>
    <StepContent>
    <div>
      Start your development server with one of the following commands:
      <CodeTabs tabs={{
        'npm':{
          syntax:'npm run dev'
        },
        'pnpm':{
          syntax:'pnpm dev'
        },
        'yarn':{
          syntax:'yarn run dev'
        },
       }} />
      </div>
    </StepContent>
  </Step>
  <Step>
    <StepTitle>Step 5: Make Changes</StepTitle>
    <StepContent>
    <div>
      <p>If you want to make changes:</p>

      1. Edit files in the root directory <code>/</code> for global configurations.
      2. Update or create MDX files in the <code>docs</code> directory to modify or add documentation.
      3. Update or create MDX files in the <code>docs</code> directory to modify or add documentation.

          <FolderTree className="overflow-hidden p-2">
            <Folder element="docs" defaultOpen={true}>
              <File>
                <p>page1.mdx</p>
              </File>
              <File>
                <p>page2.mdx</p>
              </File>
            </Folder>
          </FolderTree>
      4. Refer to the existing documentation to see how things work.  
      
      </div>
    </StepContent>
  </Step>
</Steps>
